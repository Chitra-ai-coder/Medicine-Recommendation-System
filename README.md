Medicare is a full-stack, AI-powered healthcare application that predicts potential medical conditions based on user-inputted symptoms. By leveraging a custom-trained Random Forest Classifier, it provides mathematically balanced differential diagnoses, along with targeted recommendations for medications, diets, precautions, and workout routines.

---

## KEY FEATURES

* **Advanced Symptom Analysis:** Employs a high-accuracy Random Forest model trained on thousands of clinical dataset variations to evaluate complex symptom combinations.
* **Dynamic Severity Weighting:** The training algorithm calculates a "Danger Score" for severe diseases (e.g., Typhoid, Malaria) to naturally balance voting power and solve the Machine Learning "Subset Dominance" problem.
* **Minimum Symptom Thresholds:** Enforces strict probabilistic feature generation during training, ensuring severe diseases are never falsely predicted from a single generic symptom.
* **Organic Synonym Mapping:** Automatically maps colloquial terms (e.g., "Loose Motion") to clinical dataset vocabulary without relying on fragile hardcoded rules.
* **Zero-Jank UI Architecture:** Features a premium, glassmorphism-inspired interface utilizing `requestAnimationFrame` for buttery-smooth, 60-FPS DOM manipulation.
* **Client-Side Report Generation:** Seamlessly converts diagnostic results into a downloadable, formatted PDF using `html2pdf.js`, automatically suspending CSS animations to ensure a pristine text render.

## HOW TO RUN THE PROJECT LOCALLY

Because the generated Machine Learning model (`model.pkl`) is highly complex and exceeds standard GitHub file size limits, it is explicitly ignored in version control using `.gitignore`. **You must build the AI's brain locally before starting the server.**

### 1. CLONE THE REPOSITORY
```bash
git clone [https://github.com/yourusername/medicare-ai.git](https://github.com/yourusername/medicare-ai.git)
cd medicare-ai
```

### 2. INSTALL DEPENDENCIES
It is highly recommended to use a Python virtual environment.
```bash
pip install -r requirements.txt
```

### 3. TRAIN THE MACHINE LEARNING MODEL
Run the data-engineering script to clean the datasets, apply dynamic severity weights, and compile the Random Forest model. This will generate a `model.pkl` file in your root directory.
```bash
python train_model.py
```

### 4. START THE APPLICATION
Launch the Flask server to bridge the ML model with the frontend interface.
```bash
python app.py
```
Open your web browser and navigate to `http://127.0.0.1:5000`.
